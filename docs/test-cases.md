## P1 — 翻牌逻辑

| ID | 用例名 | 前置条件 | 操作步骤 | 预期结果 |
|:---|:---|:---|:---|:---|
| P1-01 | 翻到数字牌 0 | deck 中有 0 | 玩家在自己的回合点击 GO! | 翻到 value=0 的牌，加入手牌（数字 0 不计分） |
| P1-02 | 翻到数字牌 5 | deck 中有 5 | 玩家翻牌 | 加入 value=5 的牌，手牌得分 +5 |
| P1-03 | 翻到数字牌 12 | deck 中有 12 | 玩家翻牌 | 加入 value=12 的牌，手牌得分 +12 |
| P1-04 | 重复数字牌爆牌 | 手牌中已有 value=3 的牌 | 玩家再次翻到 value=3 | 服务器返回 bust，客户端播放爆牌动画，手牌清空 |
| P1-05 | 重复但手牌有复活牌 | 手牌已有 value=2 + 1 张复活牌 | 玩家再次翻到 value=2 | 自动消耗复活牌抵消爆牌，手牌保留 |
| P1-06 | 复活牌消耗后不复原 | P1-05 后 | 检查手牌 | 被消耗的复活牌从手牌中移除，其余牌保留 |
| P1-07 | 多张复活牌改为上限1张 | 手牌已有 1 张复活牌 | 玩家翻到第二张复活牌 | 触发目标选择，不能加入第二张 |
| P1-08 | 翻到复活牌本身 | deck 中有复活牌 | 玩家翻牌 | 复活牌正常加入手牌 |
| P1-09 | 翻牌动画完整执行 | 玩家在自己回合 | 点击 GO! → 等待动画 | 5 阶段动画依次执行：showing_back(200ms) → flipping(800ms) → enlarged(800ms) → entering_hand(800ms) → idle |
| P1-10 | 翻牌过程中 DISABLED 状态 | 翻牌动画播放中 | 观察 GO/STOP 按钮状态 | 翻牌动画期间两个按钮都 disabled |
| P1-11 | 连续翻牌直到爆牌 | 手牌有 3 张不同数字牌 | 不断翻牌直到重复 | 手牌逐渐增加后爆牌清空，手牌归零 |
| P1-12 | 爆牌后视野切换 | 玩家 A 爆牌后 | 观察对手视角 | 爆牌玩家手牌清空，对手视角切换到下一玩家 |
| P1-13 | 爆牌玩家确认 | 玩家 A 爆牌 | 检查 hasBusted 和 isOut 标记 | hasBusted=true 且 isOut=true |
| P1-14 | 牌堆不足时继续翻牌 | deck.length = 3 | 玩家继续翻牌直到 deck 为空 | deck 为空时从 discard 补充新牌堆 |
| P1-15 | 牌堆和弃牌堆同时为空 | deck + discard 都空（极端情况） | 玩家尝试翻牌 | 返回失败或游戏结束 |
| P1-16 | 牌堆悬浮动画 | 玩家回合且牌堆可点击 | 观察牌堆 | 牌堆有 hover scale(1.05)，下方显示「👆 点击翻牌」 |
| P1-17 | 七连翻触发（Flip 7） | 玩家纯数字牌 ≥ 7 张 | 继续翻牌直到纯数字牌达到 7 张 | 触发七连翻庆祝动画 + +15 奖励分 + 手牌清空 + 回合结束（注：功能牌如复活牌不计入七连翻） |
| P1-17a | 七连翻：全员得分（官方规则） | A 即将触发 flip7；B/C 各有手牌 | A 翻到第 7 张纯数字牌 | **所有未出局玩家**获得各自手牌分数，触发者额外 +15 奖励。例如 A 手牌 {1-7} 得 43 分，B 手牌 {3,5} 得 8 分，C 手牌 {2} 得 2 分 |
| P1-17b | 七连翻：全员手牌清空 | 多人游戏，即将触发 flip7 | 触发 flip7 | 所有玩家手牌进入弃牌堆、isOut=true、全员结算 |
| P1-18 | 七连翻计分（单人） | 手牌 7 张纯数字牌 (1+2+3+4+5+6+7=28)，其他玩家无手牌 | 触发七连翻 | 触发者 score = 28 + 15(bonus) = 43；其他无手牌玩家得 0 分 |
| P1-19 | 七连翻后检查获胜 | 玩家 A 已有 190 分后触发七连翻 | 触发七连翻 | 总分 = 226 ≥ 200，游戏结束 A 获胜 |
| P1-20 | 6 张数字 + 1 张复活不算七连翻 | 手牌 6 数字 + 1 复活 | 翻第 7 张（触发检查） | 不触发七连翻，因为纯数字牌只有 6 张 |
| P1-21 | 7 张数字 + 1 张复活算七连翻 | 手牌 7 数字 + 1 复活 | 翻第 7 张数字牌 | 触发七连翻，因为纯数字牌有 7 张 |
| P1-22 | 七连翻中复活牌不计分 | 手牌 7 张含 1 张复活牌 | 触发七连翻 | score = 所有数字牌之和 + 15(bonus)，复活牌 value 不计分 |
| P1-23 | 翻牌后端广播 | 任意玩家翻牌后 | 观察另一玩家屏幕 | 收到 state_sync 后更新手牌/分数/轮次信息 |
| P1-24 | 翻到牌数为上限 99 | deck.length > 99 | 观察牌堆数量标签 | 显示上限 99 |
| P1-25 | 新牌飞入动画 | 玩家翻牌获得新牌 | 观察新牌出现方式 | 新牌显示 card-fly-in 飞入效果 |

## P2 — 计分系统

| ID | 用例名 | 前置条件 | 操作步骤 | 预期结果 |
|:---|:---|:---|:---|:---|
| P2-01 | 基础数字牌累加 | 手牌：{1, 3, 5} | STOP 结算 | score = 1 + 3 + 5 = 9 |
| P2-02 | 单张数字牌 | 手牌：{7} | STOP 结算 | score = 7 |
| P2-03 | 数字 0 不计分 | 手牌：{0, 5} | STOP 结算 | score = 0 + 5 = 5 |
| P2-04 | 多张同数字牌（不同值可共存） | 手牌：{2, 2, 3}（通过复活牌保留） | STOP 结算 | score = 2 + 2 + 3 = 7 |
| P2-05 | 爆牌后 0 分 | 玩家爆牌 | 检查爆牌后 history 条目 | scoreGained=0, isBust=true |
| P2-06 | 七连翻 +15 奖励 | 手牌 7 张基础牌 (1+2+3+4+5+6+7=28) | 触发七连翻 | score = 28 + 15(bonus) = 43 |
| P2-07 | 加倍牌效果（FLAG ON 时） | 手牌：{3, 5, double} | STOP 结算 | score = (3 + 5) × 2 = 16（加倍牌先翻倍数字牌，但此case无分数调整牌） |
| P2-07a | 分数调整牌：{5, +4} = 9 分 | FLAG ON | STOP 结算 | (5) + (4) = 9（数字牌 + 分数调整牌，无加倍） |
| P2-07b | 加倍 × 分数调整（官方规则） | 手牌：{3, 5, +4, double} | STOP 结算 | (3+5) × 2 + 4 = 20（先加倍数字牌，再加调整牌，调整牌不被翻倍） |
| P2-07c | 官方 case：{0,3,4,5,+4,x2,9,10,11} | 触发七连翻 | STOP 结算 | (0+3+4+5+9+10+11) × 2 + 4 + 15(七连翻) = 103 |
| P2-08 | 加倍牌 + 七连翻 | 手牌 7 张含 double (1+2+3+4+5=15, ×2=30) | 触发七连翻 | (1+2+3+4+5) × 2 + 15(bonus) = 45 |
| P2-09 | 冻结牌目标计分（FLAG ON 时） | 手牌：{freeze}，目标手牌 {2, 4} | 使用冻结牌选择目标 | 目标 score = 2 + 4 = 6，目标 isOut=true |
| P2-09a | 翻到冻结牌 → pending_action | FLAG ON, 有其他玩家存活 | 翻到 ❄️ | 进入 pending_action 状态，等待选择目标玩家 |
| P2-09b | 冻结目标结算（结算型） | 处于冻结目标选择 | 选择目标 B | B score=手牌总和, isOut=true, B 手牌进弃牌堆, 触发者自由回合继续 |
| P2-09d | 冻结后 RoundHistory 显示 | 玩家 B 被冻结 | 检查 RoundHistory | B 的格子显示实际得分（如"+8"），不能是"（−）栏"；history 中 actions=["freeze"], scoreGained=手牌总和 |
| P2-09c | 唯一存活玩家翻冻结牌 → 自动结算 | FLAG ON, 其他玩家均已 isOut | 翻到 ❄️ | 无目标可用 → 直接结算自己的手牌（score+手牌清空+isOut=true），冻结牌进弃牌堆 |
| P2-10 | 分数颜色区间 | 玩家分数不同 | 观察分数显示颜色 | <100 金色 / 100-149 黄色 / >=150 红色 |
| P2-11 | 手牌得分颜色 | 手牌得分区间 | 观察手牌得分 | <30 绿色 / 30-49 金色 / >=50 黄色 |
| P2-12 | 得分 Toast 显示 | 任意玩家翻牌得分后 | 观察 Toast | 显示「昵称 +X 分」，绿色，2500ms 后消失 |
| P2-13 | 爆牌 Toast | 玩家爆牌 | 观察 Toast | 显示「昵称 爆牌 +0 分」，红色 |
| P2-14 | 分数累计正确性 | 多轮游戏 | 每轮结束核对总分 | 每轮得分 = 前轮总分 + 本轮得分，计算一致 |

| P3-16 | 游戏结束后禁止操作 | phase=ended | 尝试点击 GO!/STOP | 按钮不显示，无法操作 |

## 附录 B — 边界场景修正说明

部分用例（P3/P4 类）依赖严格的模拟环境：

- **断线超时 / 重连** — 需要手动关闭 WebSocket 或用 DevTools 模拟断网
- **端口冲突** — 需要手动占用 3001 端口后启动
- **多端操作冲突** — 需要用两个浏览器同时操作并引入人工延迟

这些用例建议在专项网络测试阶段手工执行，或留到集成测试环境中自动执行。
