# Flip 7 — 产品需求文档（PRD）

**版本：** V0.5（增强原型）
**更新日期：** 2026-07-04
**状态：** 已立项 / 原型已验证
**目标平台：** 微信小游戏（MVP 阶段可先以 H5 原型验证）

---

## 1. 产品概述

### 1.1 产品定位
一款 2 人异步对战的数字卡牌策略小游戏，核心乐趣在于"风险评估 + 叫停时机"的心理博弈。

### 1.2 核心循环
```
翻开卡牌 → 检查是否重复 → 重复则判负 / 不重复则入牌 → 手牌达7张触发Flip7
→ 未达7张则继续翻 or 叫停 → 结算 → 轮换对手
```

### 1.3 胜利条件
任一玩家累计总分 **≥ 200 分**即获胜。

## 3. 牌组构成

### 3.1 数字牌（79 张）
| 数字 | 数量 |
|------|------|
| 0 | 1 张 |
| 1 | 1 张 |
| 2 | 2 张 |
| 3 | 3 张 |
| 4 | 4 张 |
| 5 | 5 张 |
| 6 | 6 张 |
| 7 | 7 张 |
| 8 | 8 张 |
| 9 | 9 张 |
| 10 | 10 张 |
| 11 | 11 张 |
| 12 | 12 张 |

### 3.2 特殊牌（6 张）
| 牌名 | 数量 | 效果 |
|------|------|------|
| +2 | 1 张 | 本轮暂存分数 +2 |
| +4 | 1 张 | 本轮暂存分数 +4 |
| +6 | 1 张 | 本轮暂存分数 +6 |
| +8 | 1 张 | 本轮暂存分数 +8 |
| +10 | 1 张 | 本轮暂存分数 +10 |
| x2 | 1 张 | 本轮手牌牌面值总和 ×2 |

**牌组总计：85 张**

### 3.3 特殊牌规则
- 特殊牌**不会与手牌中的数字重复**（因为它们不是数字）
- 翻到特殊牌时**总是成功**，直接应用效果
- **+N 卡牌**：直接将本轮暂存分数增加 N
- **x2 卡牌**：将当前手牌牌面值总和 ×2 作为本轮暂存分数
  - 注意：x2 是基于**当前手牌牌面值总和**，不是基于已累加的暂存分数
  - 示例：手牌 [3, 5, 7]，翻到 x2 → 暂存分数 = (3+5+7) × 2 = 30

### 3.4 牌堆状态
- `unused`：当前可供翻开的牌堆（优先使用）
- `discard`：已翻出的牌、判负/叫停后回收的手牌
- 当 `unused` 为空时，自动将 `discard` 洗牌后变为新的 `unused`
- 每局开始前，重新生成一副完整的 85 张牌并洗牌

## 4. 核心规则

### 4.1 回合流程
1. 轮到当前玩家时，界面进入 **playing** 状态
2. 玩家选择 **Go** 或 **Stop**
3. 若选择 **Go**：翻开 `unused` 顶部的牌
4. 若选择 **Stop**：本轮结算，手牌移入 `discard`，分数累加到总分

### 4.2 翻牌规则
1. **翻开卡牌**（弹窗展示 0.5 秒）
2. **检查手牌中是否有重复**（仅数字牌参与判断）
   - 重复 → 判负，手牌移入 `discard`，切换玩家
3. **不重复 → 卡牌放入手牌区域**
4. **检查手牌数量是否达到 7 张**
   - 达到 → 触发 Flip 7 奖励
5. **未达到 7 张 → 继续翻牌**
- **特殊牌**：总是成功，直接应用效果

### 4.3 叫停规则
- 玩家可在任意时机点击 **Stop**
- 手牌全部移入 `discard`
- 本轮暂存分数累加到总分
- **允许空手牌 Stop**，此时获得 0 分

### 4.4 Flip 7 奖励
- 当玩家**手牌数量达到 7 张**时，触发 Flip 7
- 奖励：**额外 +15 分**（叠加到本轮得分）
- 效果：本轮立即结束，手牌移入 `discard`，分数累加到总分
- 下一轮由 **触发者继续先手**

### 4.5 判负处理
- 翻到重复牌后，**立即自动判负**
- 手牌全部移入 `discard`，本轮得分归 0
- 自动切换玩家继续本轮；若全部判负则直接开始新一轮

## 5. 计分与胜利

### 5.1 分数计算
- 每张牌的点数 = 牌面值（0 牌得 0 分）
- 特殊牌：+N 直接加分，x2 将手牌总和翻倍
- 本轮得分 = 手牌牌面值总和 + 特殊牌效果 + Flip 7 奖励（如有）
- 总分 = 各轮得分累加

### 5.2 胜利条件
- 任一玩家总分 **≥ 200 分**时，立即获胜
- 游戏进入 **ended** 状态，展示胜利弹窗

## 8. 系统架构

### 8.1 MVP 功能流程
```
创建房间 → 生成分享卡片 → 对手加入 → 回合开始 → Go/Stop → 结算并锁定 → 通知对方回合
```

### 8.2 核心组件
- **游戏核心**：2 人异步对战、共享公开牌池、每回合 Go/Stop、首个 200 分获胜
- **前端层**：微信小游戏 / H5 原型、Canvas/DOM 渲染、本地回合状态管理
- **云服务层**：房间管理、玩家数据、牌池快照、回合锁

### 8.3 关键设计决策
- 异步房间采用"谁操作谁写入最新状态"的单一可信源模式
- 同步只依赖 roomId，避免复杂用户关系
- 建议原型阶段优先做"本地原型"验证手感，再切到云开发联调

### 8.4 待决策项
- 房间最大回合数：无上限？还是限制 50 局后强制结算？
- 掉线超时：对手超时未操作，是判负还是直接接管？
- 牌堆源：每局重新洗牌，还是房间内共用一个牌堆直到打完？

## 9. 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 牌堆 `unused` 为 0 | 自动将 `discard` 洗牌后补充到 `unused` |
| 牌堆 `unused` 和 `discard` 均为 0 | 提示"牌堆耗尽"，只能 Stop 结算 |
| 空手牌 Stop | 允许，获得 0 分 |
| 翻到重复牌 | 判负，手牌清空，等待手动点"结束回合" |
| 翻到特殊牌 | 总是成功，直接应用效果 |
| Flip 7 时总分已达 200 | 结算后立即结束游戏，触发者获胜 |
| x2 时手牌为空 | 暂存分数 = 0 × 2 = 0 |
| 连续多轮后总分累加 | 正常累加，不因轮换丢失 |
